Part 1: Goals, Philosophy & Methodology

Why crafting good CSS is difficult

  • Global Scope
    • Style rules applied to any element matched by selectors
    • Can be difficult to force styles to not be applied
  • Specificity is confusing and hard to track, particularly among several developers
    • Nothing in the dev environment that proactively uses specificity
    • No IDE tools advise about specificity in the coding environment
    • No IDE tools track selectors located in separate sheets that may have collisions
    • Browser dev tools display style overrides, but don’t provide specificity breakdown
  • CSS has a very shallow set of functional requirements, so it is easy to write terrible code that still “works"
  • SASS adds additional features (nesting, variables, mixins, extends, functions) that make best practices more elusive
